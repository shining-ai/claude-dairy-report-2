# 営業日報システム テスト仕様書

## テスト方針

| 種別 | 目的 | 対象 |
|------|------|------|
| 単体テスト | 各関数・モジュールの正常動作を検証 | バリデーション、ビジネスロジック |
| 結合テスト | APIエンドポイントのリクエスト〜レスポンスを検証 | 全APIエンドポイント |
| E2Eテスト | ユーザー操作シナリオを通じた動作を検証 | 主要ユースケース |

## E2Eテストシナリオ

### E2E-001: 営業担当者による日報提出フロー

```
1. sales ユーザーでログイン
2. 日報を新規作成（訪問記録2件、problem・plan を入力）
3. ステータスが draft であることを確認
4. 日報を更新（plan を修正）
5. 日報を提出
6. ステータスが submitted に変わることを確認
7. 提出後は編集できないことを確認（403）
```

### E2E-002: 上長による日報確認・コメントフロー

```
1. manager ユーザーでログイン
2. 日報一覧で submitted 状態の日報を確認
3. 日報詳細を開き、コメントを追加
4. 日報を確認済みにする
5. ステータスが reviewed に変わることを確認
```

### E2E-003: 顧客マスタ登録から日報作成までのフロー

```
1. manager ユーザーで顧客を新規登録
2. sales ユーザーでログイン
3. 日報作成画面で、登録した顧客が選択肢に表示されることを確認
4. その顧客を使って訪問記録を作成し、日報を提出
```
