# 営業日報システム 要件定義書

## システム概要

営業担当者が日々の顧客訪問記録と課題・翌日計画を報告し、上長がフィードバックを行う営業日報管理システム。

---

## 機能要件

### 1. ユーザー管理
- ユーザーは「営業（sales）」と「上長（manager）」のロールを持つ
- ログイン認証機能

### 2. 日報管理
- 営業担当者が日付ごとに1件の日報を作成・編集・提出できる
- 同一営業・同一日付での日報の二重登録を防止する
- 日報のステータス管理：`draft`（下書き）→ `submitted`（提出済み）→ `reviewed`（確認済み）

### 3. 訪問記録管理
- 1つの日報に対して、複数の顧客訪問記録を追加可能
- 各訪問記録は「顧客（顧客マスタから選択）」と「訪問内容（フリーテキスト）」を持つ
- 訪問記録の表示順を管理できる

### 4. Problem / Plan 管理
- 日報に「今の課題・相談（Problem）」と「明日やること（Plan）」を記入できる
- 上長は日報に対してコメントを追加できる

### 5. マスタ管理
- **顧客マスタ**：顧客情報の登録・編集・削除
- **営業マスタ（ユーザー管理）**：ユーザー情報の登録・編集・削除

---

## 非機能要件

| 項目 | 内容 |
|------|------|
| 認証・認可 | ロールベースアクセス制御（営業 / マネージャー） |
| データ整合性 | `(user_id, report_date)` にユニーク制約を付与し二重登録を防止 |
| 操作性 | 訪問記録の並び順を変更可能（sort_order） |
